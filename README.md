# game-opener

слушает микрофон и **запускает программу, когда слышит звук открывающейся
банки** (то самое «пшш / щёлк / шипение»). при срабатывании проигрывается звук
`assets/tank.wav` и открывается заданная программа — идея в том, чтобы **запускать
игру** (например, World of Tanks): открыл банку — игра пошла.

> работает на **Windows, macOS и Linux (Ubuntu)**. захват микрофона и
> воспроизведение звука идут через `sounddevice` (кроссплатформенно), команда
> запуска игры задаётся под свою ОС.

значения по умолчанию:
- **Windows** — играет `assets/tank.wav` и запускает **World of Tanks**
  (`C:\Games\World_of_Tanks_EU\WorldOfTanks.exe`; WGC поднимается в фоне);
- **macOS** — играет `assets/tank.wav` и открывает **Google Chrome**;
- **Linux** — играет `assets/tank.wav`; что запускать — задаёшь через
  `CAN_LAUNCH_PATH` / `CAN_LAUNCH_CMD` (см. ниже).

## как это работает

у звука открывающейся банки нет одной стабильной метки — в зависимости от банки
и микрофона он «размазан» по классам `Glass / Chink / Coin / Keys / Crack /
Hiss`, и те же метки срабатывают на ключи, монеты и посуду. поэтому вместо меток
используется **маленький классификатор, обученный на эмбеддингах YAMNet**:

```
микрофон → YAMNet (эмбеддинг 1024-d на каждые ~0.5с) → обученный MLP → P(банка)
        → срабатывание, когда P(банка) ≥ порога на 2 из последних 3 окон
        → звук + запуск программы (с кулдауном)
```

правило «2 из 3 окон» (временное сглаживание) — это то, что даёт стабильность:
одиночный щелчок (клавиша, мышь) не проходит, а реальное ~1-секундное событие
банки — проходит.

обученная модель уже лежит в `models/`, так что детектор **запускается сразу**,
без обучения.

действие при срабатывании (проиграть звук + запустить игру) вынесено в отдельный
модуль `actions.py` — **без импорта TensorFlow**. это нужно, чтобы ручной триггер
(`hotkey_trigger.py`) переиспользовал ровно то же действие, но стартовал мгновенно.

### точность (по `eval.py`, честная кросс-валидация без утечки событий)

рабочая точка: **порог 0.88, срабатывание на 2 из 3 окон**.

| метрика | значение |
|---------|----------|
| распознавание банки (recall) | **~90%** |
| ложные срабатывания на набор клавиатуры (худший случай) | ~7% |
| клик мыши | ~3% |
| часы / стекло / налив воды / шаги / двери | ~0% |
| общий уровень ложных срабатываний | ~1.4% |

## установка

нужен **Python 3.12** (у TensorFlow пока нет колёс под 3.14).

сначала системные зависимости — `ffmpeg` для декодирования аудио и PortAudio для
микрофона:

```bash
# macOS
brew install ffmpeg portaudio

# Ubuntu / Debian
sudo apt update && sudo apt install -y ffmpeg libportaudio2 python3.12-venv

# Windows: скачай ffmpeg с https://www.gyan.dev/ffmpeg/builds/ и добавь в PATH.
#          PortAudio ставится вместе с пакетом sounddevice, отдельно не нужен.
```

потом виртуальное окружение и пакеты:

```bash
# macOS / Linux
python3.12 -m venv .venv
source .venv/bin/activate

# Windows (PowerShell)
py -3.12 -m venv .venv
.venv\Scripts\Activate.ps1
```

```bash
# дальше одинаково на всех ОС (окружение уже активировано)
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

> все команды ниже написаны в расчёте на **активированное окружение**, поэтому
> просто `python can_listener.py`. если не активируешь — подставляй полный путь:
> `.venv/bin/python` (macOS/Linux) или `.venv\Scripts\python` (Windows).

## запуск

```bash
python can_listener.py
```

открой банку рядом с микрофоном → проиграется звук и запустится программа.
**Ctrl+C** — остановить.

доступ к микрофону:
- **Windows** — Параметры → Конфиденциальность → Микрофон → разреши доступ
  классическим приложениям.
- **macOS** — при первом запуске система попросит разрешение (Системные настройки
  → Конфиденциальность и безопасность → Микрофон), разреши.
- **Linux** — обычно доступ есть сразу; если микрофонов несколько, выбери нужный
  через `--list-devices` / `--device`.

### флаги

```bash
python can_listener.py --calibrate          # индикатор уровня микрофона и P(банка)
python can_listener.py --analyze clip.m4a   # оценить запись и выйти
python can_listener.py --threshold 0.93     # строже (меньше ложных)
python can_listener.py --cooldown 8         # пауза перед повторным срабатыванием (с)
python can_listener.py --list-devices       # список микрофонов
python can_listener.py --device 3           # выбрать микрофон по индексу
```

настройка чувствительности: если иногда срабатывает на набор текста — подними
`--threshold` до `0.92` (меньше ложных, recall ~85%). если пропускает реальные
банки — опусти к `0.85` (recall ~92%, чуть больше ложных на клавиатуру/мышь).

## ручной триггер (для съёмки)

`hotkey_trigger.py` запускает **то же самое действие** (звук + игра), но по нажатию
клавиши, а не по реальной банке. удобно для съёмки: открываешь банку/пиво в кадре,
жмёшь хоткей — и на видео это выглядит так, **будто система сама распознала банку**.

вывод в терминале **специально идентичен** настоящему детектору — тот же баннер
загрузки и та же строка `[время] Я СЛЫШУ ПИВО (P=0.9xx) -> ЗАПУСКАЮ` (P каждый раз
разный, как при реальном срабатывании). никакой пометки «это ручной запуск» нет.
при этом скрипт стартует мгновенно — без TensorFlow и без микрофона.

```bash
python hotkey_trigger.py                 # глобальный хоткей (по умолчанию Ctrl+Alt+T)
python hotkey_trigger.py --hotkey f8     # другой хоткей
python hotkey_trigger.py --enter         # триггер по Enter в этом окне
```

- глобальный хоткей работает через пакет `keyboard` даже когда окно не в фокусе
  (можно жать с цифрового блока вне кадра).
- **Windows** — работает из коробки.
- **Linux** — пакету `keyboard` нужен **root** (`sudo`) для глобального перехвата;
  без прав используй `--enter` (триггер по Enter в окне).
- если пакет `keyboard` не установлен — скрипт сам переключится в режим Enter.

## как поменять запускаемую программу (путь запуска игры)

что запускать — определяется в `actions.py` (функция `launch_game()`), приоритет:

1. `CAN_LAUNCH_CMD` — **любая shell-команда**, выполняется как есть (на любой ОС);
2. иначе значение по умолчанию под ОС:
   - **Windows** — открывает exe из `CAN_LAUNCH_PATH` (по умолчанию путь к
     `WorldOfTanks.exe`, см. `WOT_EXE` в `actions.py`);
   - **macOS** — `open -a` для приложения из `CAN_LAUNCH_PATH` (по умолчанию
     Google Chrome);
   - **Linux** — `xdg-open` для `CAN_LAUNCH_PATH`, если он задан.

примеры — запустить игру (например, танки):

```bash
# Windows (PowerShell): свой путь к exe
$env:CAN_LAUNCH_PATH="C:\Games\World_of_Tanks_EU\WorldOfTanks.exe"; python can_listener.py

# любая ОС: произвольная команда запуска
CAN_LAUNCH_CMD="open steam://rungameid/444200" python can_listener.py     # macOS
CAN_LAUNCH_CMD="xdg-open steam://rungameid/444200" python can_listener.py # Linux
```

открыть что-то другое, не игру (тот же механизм):

```bash
CAN_LAUNCH_CMD="open -a Spotify" python can_listener.py            # macOS
CAN_LAUNCH_CMD="xdg-open https://youtube.com" python can_listener.py  # Linux
```

чтобы задать программу **навсегда** — поменяй `WOT_EXE` (Windows) или ветки
`launch_game()` в `actions.py` под свою ОС.

> подсказки: на Windows exe запускается через `os.startfile` (отдельным
> процессом); на macOS `open -a` ищет приложение по имени в `/Applications`;
> на Linux укажи исполняемый файл/скрипт или URI-схему для `xdg-open`.

## как заменить голос/звук озвучки

при срабатывании проигрывается `assets/tank.wav`. заменить можно двумя способами.

способ 1 — подменить файл. положи любой свой звук вместо `assets/tank.wav`
(WAV играет сразу; для `.ogg/.opus/.flac` поставь опциональный пакет `soundfile`).

способ 2 — указать другой файл переменной `CAN_SOUND`:

```bash
CAN_SOUND=/path/to/sound.wav python can_listener.py
```

сгенерировать свою фразу через TTS, под каждую ОС свой инструмент:

- **macOS** — скрипт `make_voice.sh` (системный голос `say`, голос Milena, ru_RU,
  + обработка ffmpeg по высоте/темпу). пиши результат в свой файл и передавай его
  через `CAN_SOUND`, либо перезапиши `assets/tank.wav`:

  ```bash
  ./make_voice.sh "твоя фраза здесь"     # запишет assets/voice.wav
  CAN_SOUND=assets/voice.wav python can_listener.py
  ```

- **Linux** — через `espeak-ng`:

  ```bash
  sudo apt install -y espeak-ng
  espeak-ng -v ru -w assets/tank.wav "ну наконец-то, запускаю"
  ```

- **Windows** — через встроенный SAPI (PowerShell):

  ```powershell
  Add-Type -AssemblyName System.Speech
  $s = New-Object System.Speech.Synthesis.SpeechSynthesizer
  $s.SetOutputToWaveFile("assets\tank.wav")
  $s.Speak("ну наконец-то, запускаю")
  $s.Dispose()
  ```

> `make_voice.sh` — bash-скрипт с голосом `say`, поэтому **только для macOS**.
> на Windows/Linux пользуйся способом 1 или командами выше.

выключить звук совсем — переменной `CAN_NO_VOICE=1`:

```bash
CAN_NO_VOICE=1 python can_listener.py
```

## переобучение / оценка

```bash
python train.py    # скачает ESC-50 (~600MB, один раз), переобучит, запишет models/
python eval.py     # напечатает таблицу recall против «путаников»
```

добавь свои записи открывания банки в `references/` (любой аудио/видео-файл) и
переобучи — записи с **твоего** микрофона самые ценные положительные примеры.
сами клипы в репозиторий не коммитятся (см. `.gitignore`); публикуется только
обученная модель в `models/`.

### собрать больше обучающих клипов из интернета

```bash
python collect_web_positives.py       # поиск через yt-dlp + автофильтр
```

ищет на YouTube ролики открывания банки и оставляет только чистые (отбрасывая
озвученные/музыкальные через YAMNet). отобранные клипы попадают в
`references_web/` и автоматически подхватываются `train.py`. лицензии: скачанное
— только для *локального обучения* (в гит не попадает); публикуемый артефакт —
веса модели.

## переменные окружения

| переменная | что делает |
|------------|------------|
| `CAN_SOUND` | путь к звуку срабатывания (по умолчанию `assets/tank.wav`) |
| `CAN_NO_VOICE=1` | не проигрывать звук |
| `CAN_LAUNCH_PATH` | Windows: путь к exe; macOS: имя приложения для `open -a`; Linux: цель для `xdg-open` |
| `CAN_LAUNCH_CMD` | выполнить эту shell-команду вместо запуска по умолчанию (любая ОС) |

## файлы

| файл | назначение |
|------|------------|
| `can_listener.py` | детектор (рантайм): слушает микрофон и по банке дёргает действие |
| `actions.py` | действие при срабатывании (звук + запуск игры), без TensorFlow |
| `hotkey_trigger.py` | ручной триггер для съёмки — выглядит как настоящее срабатывание |
| `train.py` | сборка датасета и обучение `models/can_model.keras` |
| `eval.py` | таблица оценки рабочей точки |
| `collect_web_positives.py` | поиск через yt-dlp + автофильтр YAMNet → `references_web/` |
| `make_voice.sh` | генерация озвучки через `say` (только macOS) |
| `models/` | обученный классификатор + `meta.json` (порог, сглаживание) |
| `references/` | положительные клипы (твои записи; в гит не коммитятся) |
| `assets/tank.wav` | звук, который проигрывается при срабатывании |
