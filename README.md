# game-opener

слушает микрофон и **запускает программу, когда слышит звук открывающейся
металлической банки** (то самое «пшш / щёлк / шипение» газировки).

при срабатывании проигрывается смешная озвучка и открывается заданная программа
— по умолчанию браузер, но идея в том, чтобы **запускать игру** (например,
World of Tanks): открыл банку — игра пошла.

> работает одинаково на **macOS, Windows и Linux (Ubuntu)**. озвучка проигрывается
> через `sounddevice` (кроссплатформенно), команда запуска задаётся под свою ОС.

---

## как это работает

у звука открывающейся банки нет одной стабильной метки — в зависимости от банки
и микрофона он «размазан» по классам `Glass / Chink / Coin / Keys / Crack /
Hiss`, и те же метки срабатывают на ключи, монеты и посуду. поэтому вместо меток
используется **маленький классификатор, обученный на эмбеддингах YAMNet**:

```
микрофон → YAMNet (эмбеддинг 1024-d на каждые ~0.5с) → обученный MLP → P(банка)
        → срабатывание, когда P(банка) ≥ порога на 2 из последних 3 окон
        → озвучка + запуск программы (с кулдауном)
```

правило «2 из 3 окон» (временное сглаживание) — это то, что даёт стабильность:
одиночный щелчок (клавиша, мышь) не проходит, а реальное ~1-секундное событие
банки — проходит.

обученная модель уже лежит в `models/`, так что детектор **запускается сразу**,
без обучения.

### точность (по `eval.py`, честная кросс-валидация без утечки событий)

рабочая точка: **порог 0.88, срабатывание на 2 из 3 окон**.

| метрика | значение |
|---------|----------|
| распознавание банки (recall) | **~90%** |
| ложные срабатывания на набор клавиатуры (худший случай) | ~7% |
| клик мыши | ~3% |
| часы / стекло / налив воды / шаги / двери | ~0% |
| общий уровень ложных срабатываний | ~1.4% |

## установка

нужен **Python 3.12** (у TensorFlow пока нет колёс под 3.14).

сначала системные зависимости — `ffmpeg` для декодирования аудио и PortAudio для
микрофона:

```bash
# macOS
brew install ffmpeg portaudio

# Ubuntu / Debian
sudo apt update && sudo apt install -y ffmpeg libportaudio2 python3.12-venv

# Windows: скачай ffmpeg с https://www.gyan.dev/ffmpeg/builds/ и добавь в PATH.
#          PortAudio ставится вместе с пакетом sounddevice, отдельно не нужен.
```

потом виртуальное окружение и пакеты:

```bash
# macOS / Linux
python3.12 -m venv .venv
source .venv/bin/activate

# Windows (PowerShell)
py -3.12 -m venv .venv
.venv\Scripts\Activate.ps1
```

```bash
# дальше одинаково на всех ОС (окружение уже активировано)
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

> все команды ниже написаны в расчёте на **активированное окружение**, поэтому
> просто `python can_listener.py`. если не активируешь — подставляй полный путь:
> `.venv/bin/python` (macOS/Linux) или `.venv\Scripts\python` (Windows).

## запуск

```bash
python can_listener.py
```

открой банку рядом с микрофоном → запустится программа. **Ctrl+C** — остановить.

доступ к микрофону:
- **macOS** — при первом запуске система попросит разрешение (Системные настройки
  → Конфиденциальность и безопасность → Микрофон), разреши.
- **Windows** — Параметры → Конфиденциальность → Микрофон → разреши доступ
  классическим приложениям.
- **Linux** — обычно доступ есть сразу; если микрофонов несколько, выбери нужный
  через `--list-devices` / `--device`.

### ручной запуск по хоткею

кроме звука банки, триггер можно **запустить вручную глобальным хоткеем** — удобно
для теста или чтобы запустить программу по требованию, не открывая банку.

хоткей по умолчанию зависит от ОС: **Cmd + Shift + J** на macOS и **Ctrl + Alt + J**
на Windows/Linux. пока детектор запущен, нажатие этой комбинации из любого
приложения сразу проигрывает озвучку и запускает программу.

```bash
python can_listener.py --hotkey "<ctrl>+<alt>+k"    # своя комбинация
python can_listener.py --hotkey off                 # выключить хоткей
```

формат комбинации — как в [pynput](https://pynput.readthedocs.io/en/latest/keyboard.html#monitoring-the-keyboard):
модификаторы в угловых скобках (`<cmd>`, `<shift>`, `<ctrl>`, `<alt>`), обычные
клавиши — буквой.

особенности по ОС:
- **macOS** — при первом использовании система попросит разрешение
  **«Мониторинг ввода» / «Универсальный доступ»**; его надо выдать, иначе хоткей
  не сработает.
- **Linux** — глобальный хоткей работает под **X11**; на чистом Wayland
  `pynput` может не перехватывать клавиши глобально.
- **Windows** — работает из коробки.

сам детектор по звуку работает независимо от хоткея. если пакет `pynput` не
установлен или хоткей зарегистрировать не удалось — детектор просто запустится
без ручного триггера и напишет об этом в консоль.

### флаги

```bash
python can_listener.py --calibrate          # живой индикатор P(банка)
python can_listener.py --analyze clip.m4a   # оценить запись и выйти
python can_listener.py --threshold 0.93     # строже (меньше ложных)
python can_listener.py --cooldown 8         # пауза перед повторным срабатыванием (с)
python can_listener.py --list-devices       # выбрать микрофон
```

настройка чувствительности: если иногда срабатывает на набор текста — подними
`--threshold` до `0.92` (меньше ложных, recall ~85%). если пропускает реальные
банки — опусти к `0.85` (recall ~92%, чуть больше ложных на клавиатуру/мышь).

## как поменять запускаемую программу (путь запуска игры)

что запускать при срабатывании определяется в таком порядке приоритета:

1. флаг `--launch "команда"`;
2. переменная окружения `CAN_LAUNCH_CMD`;
3. значение по умолчанию под твою ОС (браузер) — задаётся функцией
   `default_launch_cmd()` в начале `can_listener.py`.

`--launch` / `CAN_LAUNCH_CMD` — это **любая shell-команда** твоей ОС, так что
запустить можно что угодно. команда запуска у каждой ОС своя.

запустить игру (например, танки):

```bash
# macOS
python can_listener.py --launch "open -a 'World of Tanks'"
python can_listener.py --launch "open steam://rungameid/444200"      # через Steam

# Windows (PowerShell)
python can_listener.py --launch 'start "" "C:\Games\WoT\WorldOfTanks.exe"'
python can_listener.py --launch 'start "" steam://rungameid/444200'  # через Steam

# Linux
python can_listener.py --launch "/path/to/wot.sh"
python can_listener.py --launch "xdg-open steam://rungameid/444200"  # через Steam
```

открыть что-то другое, не игру (тот же механизм, любая команда):

```bash
# macOS
python can_listener.py --launch "open -a Spotify"
# Windows
python can_listener.py --launch "start spotify:"
# Linux
python can_listener.py --launch "xdg-open https://youtube.com"
```

чтобы задать программу **навсегда** (не передавать каждый раз) — поменяй возврат
`default_launch_cmd()` в `can_listener.py` под свою ОС, например:

```python
def default_launch_cmd():
    if IS_MAC:
        return "open -a 'World of Tanks'"
    if IS_WINDOWS:
        return 'start "" "C:\\Games\\WoT\\WorldOfTanks.exe"'
    return "/path/to/wot.sh"
```

> подсказки по путям: на macOS `open -a` ищет приложение по имени в
> `/Applications` (либо укажи полный путь к `.app`); на Windows у команды `start`
> первый аргумент в кавычках — это заголовок окна, поэтому перед путём с
> пробелами ставится пустой `""`; на Linux укажи исполняемый файл/скрипт запуска
> или открой URI-схему через `xdg-open`.

## как заменить голос озвучки

при срабатывании проигрывается `assets/voice.wav` — смешная озвучка. заменить её
можно двумя способами.

способ 1 — просто подменить файл (работает на любой ОС). положи любой свой
`.wav` вместо `assets/voice.wav`. больше ничего делать не надо.

способ 2 — сгенерировать через TTS, под каждую ОС свой инструмент:

- **macOS** — скрипт `make_voice.sh` (использует системный голос `say`, голос
  Milena, ru_RU, + обработку ffmpeg по высоте/темпу):

  ```bash
  ./make_voice.sh "твоя фраза здесь"

  # ручки стиля: VOICE_RATE_MULT — высота (меньше 1 = ниже/«бас»), VOICE_TEMPO — темп
  VOICE_RATE_MULT=1.45 VOICE_TEMPO=0.98 ./make_voice.sh "погнали"   # писклявый вместо баса
  ```

  другой голос — поменяй `-v Milena` в `make_voice.sh` на любой установленный
  (список: `say -v '?'`).

- **Linux** — через `espeak-ng` (или `spd-say`):

  ```bash
  sudo apt install -y espeak-ng
  espeak-ng -v ru -w assets/voice.wav "ну наконец-то, запускаю"
  ```

- **Windows** — через встроенный SAPI (PowerShell), с сохранением в
  `assets/voice.wav`:

  ```powershell
  Add-Type -AssemblyName System.Speech
  $s = New-Object System.Speech.Synthesis.SpeechSynthesizer
  $s.SetOutputToWaveFile("assets\voice.wav")
  $s.Speak("ну наконец-то, запускаю")
  $s.Dispose()
  ```

> `make_voice.sh` — это bash-скрипт и голос `say`, поэтому он **только для macOS**.
> на Windows/Linux пользуйся способом 1 или командами выше.

выключить озвучку совсем — переменной `CAN_NO_VOICE=1`:

```bash
CAN_NO_VOICE=1 python can_listener.py
```

## переобучение / оценка

```bash
python train.py    # скачает ESC-50 (~600MB, один раз), переобучит, запишет models/
python eval.py     # напечатает таблицу recall против «путаников»
```

добавь свои записи открывания банки в `references/` (любой аудио/видео-файл) и
переобучи — записи с **твоего** микрофона самые ценные положительные примеры.
сами клипы в репозиторий не коммитятся (см. `.gitignore`); публикуется только
обученная модель в `models/`.

### собрать больше обучающих клипов из интернета

```bash
python collect_web_positives.py       # поиск через yt-dlp + автофильтр
```

ищет на YouTube ролики открывания банки и оставляет только чистые (отбрасывая
озвученные/музыкальные через YAMNet). отобранные клипы попадают в
`references_web/` и автоматически подхватываются `train.py`. лицензии: скачанное
— только для *локального обучения* (в гит не попадает); публикуемый артефакт —
веса модели.

## файлы

| файл | назначение |
|------|------------|
| `can_listener.py` | сам детектор (рантайм) + ручной хоткей + запуск программы |
| `train.py` | сборка датасета и обучение `models/can_model.keras` |
| `eval.py` | таблица оценки рабочей точки |
| `collect_web_positives.py` | поиск через yt-dlp + автофильтр YAMNet → `references_web/` |
| `make_voice.sh` | (пере)генерация озвучки `assets/voice.wav` (только macOS) |
| `models/` | обученный классификатор + `meta.json` (порог, сглаживание) |
| `references/` | положительные клипы (твои записи; в гит не коммитятся) |
| `assets/voice.wav` | озвучка срабатывания |
